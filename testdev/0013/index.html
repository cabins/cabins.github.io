#13 编程语言学习建议

Sun, Apr 12, 2020

B 站《测试开发工程师的日常》配套讲稿

编程语言学习方法

对于测试开发工程师来说：

  • 学会通过Quickstart来快速的掌握一门编程语言的特点
  • 用最短的时间掌握该编程语言与其他编程语言的共有特性
    • 因为你已经有了其他编程语言的基础，所以这部分没必要投入很多的精力
    • 只要掌握不同的点就好，比如Go语言的循环全部靠for循环实现
  • 通过实战，最好是项目实战进行编程语言的学习
  • 注重总结和分享
    • 工作小组内分享
    • 写博客/公众号文章
    • 开源项目
    • 视频教程录制

编程语言学习深度

对于测试开发工程师来说：

  • 掌握语言的基础知识
  • 掌握必要的进阶知识
  • 其余的在项目中按需而学
  • 知道或理解编程语言的知名框架，用到的时候再学
  • 对于测试开发来说，学会利用框架而非造轮子很重要（这一点与软件开发工程师很不同）
  • 尽量选择大而全的框架，而非处处都要自己搭建的框架
    • 自己搭建的框架不能保证框架的质量（把Flask手动组装成Django一样的功能，其稳定性极差）
    • 自己搭建的框架不能保证最后的性能（把Flask手动组装成Django一样的功能，其性能远不如Django）

编程语言选择

对于测试开发工程师来说：

  • 最好掌握一门动态语言（快速开发脚本或验证想法）
    • Python
    • JavaScript（Node.js, TypeScript)
  • 最好掌握一门静态语言（用于大型平台或项目的开发）
    • Java
    • Go
  • 同时建议掌握Shell的基础

保持学习习惯

对于测试开发工程师来说：

  • 编程语言层出不穷，日新月异，同时又大同小异，建议保持学习的习惯来应对变幻的时代
  • 保持学习习惯的第一步，是建立并保持学习的意识
  • 当你觉得毅力受到挑战时，不要放弃学习，可以尝试换一种学习对象，之后再切换回来
